Installation & basic usage

  • topics
  • installation

REMEMBER THAT FOR NOW YOU HAVE TO USE UBUNTU >= 14.04.

Download

  • Get MakinaStates by cloning it from github

    git clone http://raw.github.com/makinacorpus/makina-states

boot-salt2.sh, the makina-states installer & manager

  • boot-salt2.sh will try to remember how you configured makina-states on each run. It stores configs in installdir/etc/makina-states

  • Short overview:

    ./boot-salt2.sh --help

  • Detailed overview:

    ./boot-salt2.sh --long-help

  • If you want to install with default options (scratch)
    as root

    ./boot-salt2.sh -C

Optional post install steps

  • Install the logrotate to rotate salt logs

    ./boot-salt2.sh -C --install-logrotate

  • Install the cron that refresh makina-states code every since and then (15min)

    ./boot-salt2.sh -C --install-crons

  • Install the salt & ansible binaries to /usr/local/bin

    ./boot-salt2.sh -C --install-links

    CLI Exemples

  • Full fledged install

    ./boot-salt2.sh -C --install-logrotate --install-crons --install-links

  • To switch on a makina-states branch, like the v2 branch in production:

    ./boot-salt2.sh -b v2

  • To sync makinastates code

    ./boot-salt2.sh -C -S

Running makinastates in highstate mode

  • To install all enabled makinastates services after having configured your pillar

    ./boot-salt2.sh -n laptop|server|lxccontainer|vm -C && \
    salt-call --retcode-passthrough state.sls makina-states.top

Upgrade

  • Upgrade will:

    • Run predefined & scheduled upgrade code
    • Update makina-states repositories in /srv/salt & /srv/makina-states
    • Update core repositories (like salt code source in /srv/makina-states/src/salt)
    • Do the highstates (salt and masterone if any)
    boot-salt2.sh -C --S  && \
    salt-call --retcode-passthrough state.sls makina-states.top
